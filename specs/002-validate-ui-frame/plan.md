# Implementation Plan: UI Frame Shell OCP — ShellManager & Region Contracts

## Summary

Introduce `ShellManager` as an Angular injectable service that acts as the sole
composition root for the UI Frame shell. Four public region contracts
(`ICentralRegionTab`, `ISidebarEntry`, `IToolbarAction`, `IBottomPanelEntry`) in
`src/app/shell/contracts/` allow any domain application to register content in
the shell's six regions without touching shell source code. ShellManager
dispatches NgRx actions to a new `shellContent` slice and registers handlers in
`CommandRegistry`. Mock reference implementations in `src/app/shell/mock-ui/`
validate that the contracts are sufficient and are bootstrapped via
`APP_INITIALIZER` in `app.config.ts`. The existing `MockUiScreenComponent` and
its `AppComponent` toggle are deleted.

## Phase 0: Research (Complete)

Resolved unknowns documented in [research.md](research.md):

1. **State slice**: New `shellContent` NgRx slice (not extending workspace/uiContext).
2. **Type in store**: `Type<unknown>` stored in NgRx state is acceptable (no strict serialization checks in project).
3. **NgComponentOutlet**: Added to `ContentAreaComponent` behind new `componentType` input.
4. **CommandRegistry**: `addToolbarAction` auto-registers handler with `commandId = 'shell.action.' + id`.
5. **ShellComponent wiring**: ShellComponent gains new selectors → passes data to child component inputs.
6. **Bootstrap**: `APP_INITIALIZER` in `app.config.ts` calling `mock-content.initializer.ts`.
7. **Duplicate IDs**: Silent ignore + `console.warn`.
